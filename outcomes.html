Course outcomes and evidence

Each of the five program outcomes mapped to the work that demonstrates it: the artifacts, their narratives, the recorded code review, and the professional self-assessment.

Course outcome Evidence in this portfolio
Outcome 1: Employ strategies for building collaborative environments that enable diverse audiences to support organizational decision making in the field of computer scienceThe databases enhancement ships a CONTRIBUTING.md with a branching strategy and pull-request template, an auto-generated OpenAPI contract that lets front-end and back-end developers build from separate repositories, and three named roles, a shelter manager, an intake clerk, and a partner-rescue auditor, each making a specific decision from one source of truth. Evidence: Databases artifact and self-assessment.
Outcome 2: Design, develop, and deliver professional-quality oral, written, and visual communications that are coherent, technically sound, and appropriately adapted to specific audiences and contextsThe recorded code review is communication for a manager rather than a compiler. Each artifact has a written narrative, a README, and header comments that explain why a decision was made. The professional self-assessment frames the whole portfolio for a hiring audience.
Outcome 3: Design and evaluate computing solutions that solve a given problem using algorithmic principles and computer science practices and standards appropriate to its solution, while managing the trade-offs involved in design choicesThe algorithms enhancement replaces an unbalanced binary search tree with a self-balancing AVL tree and adds a directed prerequisite graph with three-color depth-first cycle detection, each choice justified with Big-O analysis and an explicit trade-off discussion. The databases indexing decisions and the SQLite-versus-PostgreSQL choice in the software design artifact are evaluated the same way.
Outcome 4: Demonstrate an ability to use well-founded and innovative techniques, skills, and tools in computing practices for the purpose of implementing computer solutions that deliver value and accomplish industry-specific goalsEach artifact applies current production-grade tools correctly: React, Node, Express, SQLite, JWT and bcrypt in the software design artifact; AVL trees and three-color DFS in the algorithms artifact; FastAPI, MongoDB 7, Pydantic, and 2dsphere indexing in the databases artifact.
Outcome 5: Develop a security mindset that anticipates adversarial exploits in software architecture and designs to expose potential vulnerabilities, mitigate design flaws, and ensure privacy and enhanced security of data and resourcesSecurity is treated as a design constraint across all three artifacts: hashed passwords, pinned JWT algorithm, server-enforced role-based access, parameterized queries and schema validation, secrets kept out of source with fail-fast on missing configuration, a rate-limited login endpoint, and an audited write chokepoint with secret redaction. The narratives are explicit about the limits of each claim. See all three artifact pages and the self-assessment.

How the evidence is distributed

No single artifact carries an outcome alone. Each enhancement leads on a different core area while contributing to the others, and the self-assessment and code review tie them together. As a result, all five outcomes are demonstrated more than once across the portfolio.

  • Software design and engineering, CS 360: full-stack rebuild with a service layer, server-enforced roles, and parameterized queries.
  • Algorithms and data structures, CS 300: AVL tree, prerequisite graph, and three-color depth-first cycle detection with written complexity analysis.
  • Databases, CS 340: FastAPI and MongoDB service with schema validation, deliberate indexing, and an audited write chokepoint.
