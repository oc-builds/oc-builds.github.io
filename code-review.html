02 / Code review

A walkthrough before the enhancements

A recorded review of the original code for all three artifacts: the weaknesses in each and the plan for the enhancement that follows. It covers existing structure, functionality gaps, security shortcomings, and the changes each enhancement makes to close them.

What the review covers

The walkthrough opens each original artifact, reads through its structure, and names the specific weaknesses that the enhancement will fix. For the inventory app it is the fake login, the missing database, and the empty screens. For the course advisor it is the unbalanced tree, the unchecked prerequisites, and the lack of any defense against a circular catalog. For the animal shelter project it is the hard-coded credentials, the missing indexes, and the absence of schema validation, role separation, and an audit trail. Each segment ends with the plan for the rebuild, so the review and the enhanced code can be read against each other.
